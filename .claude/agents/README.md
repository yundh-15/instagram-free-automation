# 코인 투자 에이전트 팀

기획서([`docs/coin-agent-team-plan.md`](../../docs/coin-agent-team-plan.md))의 팀을 Claude Code 서브에이전트로 구현한 디렉터리입니다.

> **효율화 이력 (8 → 6명)**
> - ❌ `data-analyst` 제거: 시세/캔들 수집·정규화는 결정적 코드(`coin/lib/feed.mjs`)와 Upbit MCP 조회 도구의 영역. LLM을 쓰면 지연·비용·환각만 추가됨.
> - 🔀 `sentiment-analyst` → `research-analyst` 통합: 둘 다 웹리서치 기반이라 중복. 리서치 애널리스트가 펀더멘털 + 뉴스/심리를 겸함.

## 팀원 (6명)

| 파일 | 직무 | 모델 | 사고 | 핵심 책임 |
|------|------|------|------|-----------|
| `cio.md` | 최고투자책임자(CIO) | opus | High | 시그널 종합·최종 결정안 |
| `technical-analyst.md` | 기술분석 애널리스트 | sonnet | Medium | 코드 지표 해석·국면 판단·시그널 |
| `research-analyst.md` | 리서치 애널리스트 | sonnet | Low~High | 펀더멘털·온체인 + 뉴스·심리(통합) |
| `risk-manager.md` | 리스크 매니저(CRO) | opus | High | 주문 사전 검증·거부권(veto) |
| `quant-researcher.md` | 퀀트 리서처 *(비동기·선택)* | opus | High | 백테스트·전략 최적화 |
| `trading-desk.md` | 트레이딩 데스크 | haiku | Minimal | 승인된 주문 집행 |

> 데이터 수집은 에이전트가 아니라 **코드 데이터 계층 + Upbit MCP 도구**가 담당한다.

## 의사결정 흐름 (1 사이클)

```
[코드 데이터 계층 / Upbit MCP]  ──시세·캔들──┐
                                            ▼
기술분석 애널리스트 ┐                    [입력]
리서치 애널리스트   ┘──분석/점수──▶  CIO  ──결정안──▶  리스크 매니저
                                                          │ 승인
                                                          ▼
                                                  트레이딩 데스크 ──▶ 거래소
                                                          │
                                                    상태 갱신·감사 로그
퀀트 리서처 ── 비동기 백테스트 ──▶ 전략/파라미터 제안 (선택)
```

## 운영 원칙
- **역할 분리**: 분석 / 결정(CIO) / 검증(리스크 매니저) / 실행(트레이딩 데스크)을 분리한다.
- **결정적인 일은 코드로**: 데이터 수집·지표 계산은 코드, LLM은 해석·판단·종합에만.
- **리스크 매니저 거부권**: 승인 없는 주문은 실행 불가.
- **페이퍼 트레이딩 우선**: 기본값은 모의. 검증 후 소액 실거래 → 한도 확대.
- **MVP 3인**: `cio` + `risk-manager` + `trading-desk` (+ 기술분석가). 리서치·퀀트는 단계적으로.

## 호출 정책 (토큰 효율) — "에스컬레이션 온리"

원칙: **핫패스는 코드(0토큰), LLM은 필요할 때만.** 매 사이클 6명을 다 부르지 않는다.

| 에이전트 | 평상시(0토큰 코드) | LLM 호출 시점 |
|----------|-------------------|----------------|
| 데이터·지표 | 코드/MCP가 처리 | (호출 안 함) |
| 기술분석 | `indicators.mjs` 시그널 | 코드 시그널이 neutral/경계, 복잡 패턴 해석 시만 |
| 리서치 | 브리지 파일 캐시 읽기 | TTL 만료 또는 중대 이벤트 시만(저빈도) |
| CIO | `coin/agents/cio.mjs` | 신호 상충·대형 포지션·이례적 상황만 |
| 리스크 매니저 | `coin/agents/risk-manager.mjs` 게이트 | 한도 경계 모호·이례적 대형주문만 |
| 트레이딩 데스크 | 코드/Haiku 실행 | 승인된 주문이 있을 때만 |
| 퀀트 리서처 | — | 비동기 배치(주 1회 등), 실시간 루프 밖 |

**비용 절감 메커니즘**
1. **0토큰 핫패스**: 고빈도 의사결정·실행은 전부 결정적 코드. LLM 토큰 0.
2. **캐시(브리지 파일)**: 가장 비싼 리서치는 TTL 내 재호출 금지. 루프는 파일만 읽음(0토큰).
3. **에스컬레이션**: LLM은 코드가 애매할 때만 끌어올려 부른다(상시 호출 아님).
4. **모델 차등**: 저빈도·고난도 → Opus, 실행 → Haiku, 고빈도 → 코드.

> 각 에이전트 파일 하단의 `## 호출 조건 (토큰 효율)` 에 개별 트리거를 명시했다.

## 사용법
서브에이전트는 이름으로 호출합니다(예: `technical-analyst`, `risk-manager`).
실데이터 연동 시 Upbit MCP 서버를 런타임에 등록하면 조회·주문 도구를 사용할 수 있습니다.
