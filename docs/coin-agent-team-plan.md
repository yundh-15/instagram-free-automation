# 코인 투자 에이전트 팀 기획서

> 작성일: 2026-06-20
> 목적: 암호화폐(코인) 투자 의사결정·실행을 자동화하기 위한 멀티 에이전트 팀의 구성, 모델 배정, 사고 수준(thinking budget), 인원수를 정의한다.

---

## 1. 설계 원칙

1. **역할 분리(Separation of Duties)**: 분석·결정·실행·리스크 검증을 서로 다른 에이전트가 맡아 단일 에이전트의 편향·환각이 자금 손실로 직결되지 않게 한다.
2. **비용/지연 vs 추론력 균형**: 자주 도는 단순 작업은 빠르고 저렴한 모델, 드물지만 중요한 판단은 강력한 모델에 배정한다.
3. **휴먼 인 더 루프(Human-in-the-loop)**: 실제 자금 집행 전에는 사람 승인 또는 사전 정의된 자동 한도 내에서만 실행한다.
4. **감사 가능성(Auditability)**: 모든 결정에 근거 로그(데이터 스냅샷·추론 요약)를 남긴다.
5. **점진적 권한**: 초기에는 페이퍼 트레이딩(모의)으로 운영하고, 검증 후 소액 실거래 → 한도 확대 순으로 권한을 넓힌다.

---

## 2. 팀 구성 개요 (총 6 에이전트)

> **효율화(8 → 6)**: 데이터 수집은 LLM이 아니라 결정적 코드/Upbit MCP가 맡는 게 맞아 **데이터 애널리스트를 제거**했고, 웹리서치로 겹치던 **시장심리 애널리스트를 리서치 애널리스트로 통합**했다. (상세 근거: §6)

| # | 에이전트 (직무명) | 핵심 책임 | 추천 모델 | 사고 수준 | 호출 빈도 |
|---|----------|-----------|-----------|-----------|-----------|
| 1 | **최고투자책임자 (CIO)** | 전체 흐름 조율, 의견 종합, 최종 의사결정 라우팅 | Claude Opus 4.8 | High | 사이클마다 |
| 2 | **기술분석 애널리스트** | 코드가 산출한 지표(RSI, MACD, 추세) 해석·국면 판단·시그널 | Claude Sonnet 4.6 | Medium | 사이클마다 |
| 3 | **리서치 애널리스트** | 펀더멘털·온체인·토크노믹스(중장기) + 뉴스·소셜·심리(단기) 통합 | Claude Sonnet 4.6 | Low~High | 시간/일 + 이벤트 시 |
| 4 | **리스크 매니저 (CRO)** | 포지션 사이징, 손절/익절, 노출 한도, VaR 검증, 거부권 | Claude Opus 4.8 | High | 모든 주문 전 |
| 5 | **퀀트 리서처** *(비동기·선택)* | 전략 백테스트, 파라미터 튜닝, 가설 검증 | Claude Opus 4.8 | High | 비동기/배치 |
| 6 | **트레이딩 데스크** | 거래소 API 주문 실행, 체결 확인, 슬리피지 관리 | Claude Haiku 4.5 | Minimal | 주문 발생 시 |

> 데이터 수집·지표 계산은 **코드 데이터 계층(`coin/lib/`) + Upbit MCP 조회 도구**가 담당한다(에이전트 아님).
>
> **최소 운영 구성(MVP, 3~4)**: 최고투자책임자(CIO) + 리스크 매니저 + 트레이딩 데스크 (+ 기술분석 애널리스트).
> 리서치·퀀트는 단계적으로 추가한다.

## 3. 역할별 상세 정의

### 3.1 최고투자책임자 (CIO / Orchestrator)
- **책임**: 각 애널리스트 의견을 수집해 가중 종합 → 매수/매도/보류 결정안 생성 → 리스크 매니저 검증 통과 시 트레이딩 데스크에 전달.
- **모델/사고**: Opus 4.8 / High thinking. 여러 상충 신호를 종합하는 핵심 추론이라 가장 강력한 모델 배정.
- **입력**: 모든 애널리스트 출력 + 현재 포트폴리오 상태.
- **출력**: 구조화된 결정안(JSON): `{action, symbol, size, confidence, rationale, stop_loss, take_profit}`.

> **데이터 수집은 에이전트가 아님**: 시세·거래량·오더북·캔들은 결정적 코드(`coin/lib/feed.mjs`)와 Upbit MCP 조회 도구가 공급한다. LLM을 데이터 배관에 쓰면 지연·비용·환각만 늘어나므로 별도 데이터 에이전트를 두지 않는다.

### 3.2 기술분석 애널리스트 (Technical Analyst)
- **책임**: 코드가 산출한 지표(SMA/EMA/RSI/MACD)를 **해석**해 국면(추세/횡보)을 판단하고 단기 시그널·진입/청산 후보가·손절 라인을 제시. 지표 수치 계산 자체는 코드가 한다.
- **모델/사고**: Sonnet 4.6 / Medium. 규칙적이되 맥락 해석이 필요.

### 3.3 리서치 애널리스트 (Research & Market Intelligence)
- **책임**: 펀더멘털·온체인·토크노믹스(중장기)와 뉴스·소셜·공포탐욕지수(단기 심리)를 **한 사람이 통합** 분석. 정성 종합 점수(score)와 이벤트 경보를 제공.
- **모델/사고**: Sonnet 4.6 / Low~High(작업별 적응) — 펀더멘털은 깊게, 뉴스 스캔은 가볍게.
- **주의**: 외부 텍스트는 신뢰 불가 입력으로 취급(프롬프트 인젝션 방어). *과거 시장심리 애널리스트와 통합된 역할.*

### 3.4 리스크 매니저 (CRO / Risk Manager, 거부권 보유)
- **책임**: 모든 주문에 대한 **사전 검증 게이트**. 포지션 한도, 최대 손실, 상관관계 집중도, 레버리지 점검. 위반 시 **거부**.
- **모델/사고**: Opus 4.8 / High. 자금 보호의 마지막 방어선이라 최고 사양.
- **권한**: 최고투자책임자(CIO)의 결정도 거부 가능(veto).

### 3.5 퀀트 리서처 (Strategy Researcher) *(비동기·선택)*
- **책임**: 신규 전략 가설, 백테스트, 파라미터 최적화. 실시간 루프 밖에서 비동기로 동작. 실시간 의사결정 비용에 포함되지 않는 선택적 역할.
- **모델/사고**: Opus 4.8 / High. 배치성이라 비용보다 품질 우선.

### 3.6 트레이딩 데스크 (Trade Executor)
- **책임**: 검증 통과 주문을 거래소 API로 실행, 부분체결·재시도·슬리피지 처리, 결과 회신.
- **모델/사고**: Haiku / Minimal. 결정론적 실행이 핵심이라 코드 위주 + 최소 추론.

## 4. 모델 & 사고 수준 배정 요약

| 모델 | 용도 | 배정 에이전트 |
|------|------|----------------|
| **Claude Opus 4.8** | 고난도 종합·리스크·연구 | 최고투자책임자(CIO), 리스크 매니저, 퀀트 리서처(비동기) |
| **Claude Sonnet 4.6** | 균형형 분석 | 기술분석 애널리스트, 리서치 애널리스트 |
| **Claude Haiku 4.5** | 저지연·도구 호출 | 트레이딩 데스크 |
| *(코드/MCP)* | 데이터 수집·지표 계산 | `coin/lib/` + Upbit MCP (에이전트 아님) |

**사고 수준(thinking budget) 가이드**
- **Minimal**: 주문 실행(트레이딩 데스크) — 지연·비용 최소화. (데이터 수집은 코드/MCP가 처리)
- **Low~Medium**: 정형 분석(기술분석 애널리스트, 리서치의 뉴스/심리 스캔).
- **High**: 다중 신호 종합·리스크·전략(최고투자책임자(CIO), 리스크 매니저, 퀀트 리서처, 리서치의 펀더멘털 심층).

> 원칙: *호출 빈도가 높을수록 모델·사고 수준을 낮추고, 의사결정 영향이 클수록 높인다.*

## 6. 인원수(에이전트 수) 결정 근거 — 8 → 6 효율화

원칙: **결정적인 일은 코드로, LLM은 판단·해석·종합에만.** 에이전트 수를 늘릴수록 신호 다양성은 늘지만 지연·비용·조율 복잡도가 커진다. 이에 따라 초기 8명을 다음과 같이 정리했다.

- ❌ **데이터 애널리스트 제거**: 시세/캔들 수집·정규화는 결정적 코드(`coin/lib/feed.mjs`)와 Upbit MCP 조회 도구의 일이다. LLM을 데이터 배관에 두면 지연·비용·환각 위험만 추가된다. 해당 에이전트의 프롬프트조차 "판단하지 말고 코드로 처리"라고 적혀 있어 LLM일 이유가 없었다.
- 🔀 **시장심리 → 리서치 통합**: 둘 다 `WebSearch/WebFetch` 기반의 외부정보 수집·점수화로 작업 패턴이 동일. 소규모 팀에서 웹리서치 LLM 2개는 과하므로 리서치 애널리스트가 펀더멘털 + 뉴스/심리를 겸한다.
- ✅ **유지**: CIO·리스크 매니저는 "결정 vs 거부"의 권한 분리라 합치지 않는다. 기술분석가는 코드 지표의 *해석* 역할로 존속. 트레이딩 데스크는 실행 전용. 퀀트 리서처는 고유 가치(백테스트)가 있어 유지하되 비동기·선택.

**최종 6명** + 데이터/지표는 코드·MCP. **MVP 3~4**: CIO + 리스크 매니저 + 트레이딩 데스크 (+ 기술분석가).

## 7. 리스크 & 가드레일

- **자금 보호**: 리스크 매니저 거부권(veto), 일일 최대 손실(킬 스위치), 단일 종목 노출 한도.
- **모의→실거래 단계화**: 페이퍼 트레이딩으로 N주 검증 후 소액 실거래.
- **프롬프트 인젝션 방어**: 뉴스/소셜 등 외부 텍스트는 신뢰 불가 입력으로 격리.
- **휴먼 승인**: 일정 금액 이상 주문은 사람 승인 필수.
- **관측성**: 모든 결정의 입력 스냅샷·추론 요약·결과를 로깅(감사·복기용).
- **면책**: 본 기획은 자동화 설계 문서이며 투자 자문이 아니다. 암호화폐는 고위험 자산이다.
